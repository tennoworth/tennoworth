Layout mockups

12 distinct organisational schemes for the inventory dashboard. Open each in a new tab and flick between them. Same palette, same data, different ways of putting it on screen. The UI agent recommends 05 — dense dashboard; everything else is here to argue against (or with) that.

The agent's recommendation: keep the layout flat, fix the hierarchy. The page only feels heavy because everything competes at equal weight — not because anything is genuinely buried.

Shrink the stats strip to a single line, compress set picks + relic planner to a mini-card strip, let the results table become the centre of gravity. Reach for menus only when a fifth or sixth panel arrives (riven grader, Baro tracker) — then a sidebar earns its keep.

Open questions

1. Set picks + relic planner: inline mini-cards (05) or behind Cmd+K / drawer (07/04)? That's the only real menus-vs-not call.
2. Is a per-item detail view (relic sources, set membership) on the roadmap? If yes, the split-detail variant (09) becomes much stronger.
3. Mobile worth designing for, or desktop-only? If desktop-only, drop the bottom-nav variant (08).
